# /docs — orientation layer

Start here at the beginning of any session.

**Read order for a new session:** VISION → ARCHITECTURE → STATUS → WORKFLOW.

- **VISION.md** — what we're building and why; the strategy.
- **ARCHITECTURE.md** — the actual system: stack, data model, routes, gotchas, design tokens. Authored from the repo.
- **STATUS.md** — the living file: what's shipped, what's in flight, open decisions. Update it at the end of any session that ships or decides something.
- **WORKFLOW.md** — how we work (sprints, push rules, review gates).

**Authority order:** the **repo + `git log` + `CLAUDE.md` are authoritative**. These docs are a map, not the territory — if a doc disagrees with the code, the code wins and the doc gets fixed. `STATUS.md` is the only file expected to change every working session; the others change when the system or the plan changes.
